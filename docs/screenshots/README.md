# README screenshots

Marketing and README images for Penfold live here. The main [README.md](../../README.md) gallery links to these files by relative path.

## Where to put files

Add PNG or JPG screenshots directly in this folder (`docs/screenshots/`). Do not commit placeholder images — only real captures from a device or emulator.

## Current filenames (wired in README)

These are the files referenced by the v0.3.0 README gallery:

| File | What to capture |
|------|-----------------|
| `library.jpg` | Home library: colored notebook covers, folders, tag filter chips, search bar |
| `editor.jpg` | Active notebook: handwriting on ruled/grid paper, top toolbar, floating page pill |
| `page-overview.jpg` | Page overview grid with thumbnails, reorder handles, OCR badge if visible |
| `page-settings.jpg` | Page settings popup: paper theme, template, size, orientation, export |
| `settings.jpg` | Settings screen: toolbar order, backup, stroke smoothing, About |
| `supported-languages.jpg` | Language/locale picker showing all 14 supported translations |
| `dark-library.jpg` | App library in dark theme |
| `dark-editor.jpg` | Notebook editor in dark theme |
| `ocr.jpg` | Handwriting OCR: indexed page badge, search results, or convert-to-text |

When replacing a shot, keep the same filename so README links stay valid. Add new files with descriptive names and update the README table if you expand the gallery.

## Image tips

- **Aspect ratio:** Prefer tablet landscape (16:10 or 4:3) — Penfold targets Galaxy Tab–class devices.
- **Format:** PNG preferred for crisp UI text; JPG is fine for large galleries.
- **Resolution:** 1920×1200 or similar is enough; avoid upscaled or heavily compressed shots.
- **Content:** Use sample notebooks with readable ink, not personal data.
- **Consistency:** Same device theme (light vs dark) per row in the README table reads best.

After adding or renaming files, open the main README on GitHub to confirm paths and captions look right.
